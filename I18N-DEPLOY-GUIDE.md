# Guide.Ferryman 中英雙語切換（i18n v5.1）部署指南

## 這次升級做了什麼

在導覽列右側新增一顆 **「🌐 EN / 🌐 中文」** 切換鈕，全站可在「原版（中文）」與「英文版」之間自由切換：

- 語言選擇記在瀏覽器（localStorage），跨頁面、下次造訪都會記住。
- **預設仍是中文原版**，未點擊切換的訪客看到的畫面與現在 100% 相同。
- 翻譯範圍：全部 8 個前台頁面的靜態內容、導覽列、頁尾、會員彈窗（加入／升級／登入／我的會員資料）、會員閘門、限時開放橫幅、投資地圖州別面板欄位標題、`resources.html` 台灣＋美國補助全套表格、頁面 `<title>` 與表單 placeholder。
- 後台（admin/）維持中文，不受影響。

## 完全不動既有功能的做法

這次**沒有改寫任何一行既有程式邏輯**，變更只有三種：

| 檔案 | 變更 |
|---|---|
| `assets/js/i18n.js` | **全新檔案**：中英字典＋翻譯引擎（EN 模式時以精確比對置換畫面文字節點；MutationObserver 處理動態插入的彈窗、閘門、表格） |
| `assets/css/style.css` | **檔尾附加** 12 行（切換鈕樣式；不動既有任何規則） |
| 8 個 html（index / services / about / contact / insights / invest-map / trends / resources） | 各在 `</body>` 前**加 1 行** `<script src="assets/js/i18n.js" defer></script>` |

會員驗證、Supabase 讀寫、Formspree、文章渲染、地圖互動、分類篩選等功能全部原封不動；已用 jsdom 對兩種語言、8 個頁面跑過 73 項自動化檢查全數通過（含「切換分類後功能仍正常」「中文模式畫面零變動」）。

## 部署方式（GitHub 網頁上傳）

1. 進入 repo → `assets/js/` → **Upload files** → 上傳 `i18n.js`。
2. 進入 `assets/css/` → 上傳 `style.css`（覆蓋；本包內版本＝線上版＋檔尾附加）。
3. 回到 repo 根目錄 → 上傳 8 個 html 檔（覆蓋）。
4. Commit 後等 GitHub Pages 重建（約 1–2 分鐘），開啟網站點右上「🌐 EN」驗證。

> 本包內所有檔案皆以 2026/7/8 的線上版為基準製作；若您在此之後有改過線上檔案，請告訴我，我會重新以最新版合入。

## 設計上的一個重要界線：介面 vs. 資料

「英文版」翻譯的是**網站介面與靜態內容**。以下三類屬於**資料庫／資料檔內容**，在英文模式下仍顯示原文（中文），介面框架（欄位標題、按鈕、徽章）則會是英文：

1. `articles`／`hot_topics` 的文章標題、摘要、內文（每日 pipeline 產出的中文內容）
2. `invest-map` 各州 12 欄盤點**資料值**（欄位標題已翻譯）
3. 白銀／黃金會員的 premium 文章內文

若要做到「資料也全英文」，建議的下一步（可另行啟動）：

- **文章雙語化**：在每日/每週 pipeline 的 Gemini 摘要步驟加一次英文摘要，`articles`/`hot_topics` 增加 `title_en`/`summary_en` 欄位，前台依 `gfLang()` 取用——改動集中在 Python 腳本＋`main.js` 渲染處，成本低。
- **50 州資料英譯**：`investmap-data.js` 約 600 行、12 欄 × 50 州，建議做成 `investmap-data.en.js` 平行檔，逐批翻譯並沿用「查無公開來源，待查證」的標示原則。

## 維護方式

之後新增中文介面文字時，只要在 `assets/js/i18n.js` 的 `D = { ... }` 字典加一組 `"中文": "English"` 即可；含變數的句型（如「今日熱門動態 · AI」）加在 `P = [ ... ]` 樣式區。找不到對應翻譯的文字會**維持原文顯示**，不會壞版。
